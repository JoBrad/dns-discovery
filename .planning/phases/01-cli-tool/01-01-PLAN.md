<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI input to discovery engine | Untrusted domain string crosses into resolver and parsing logic |
| Discovery engine to DNS infrastructure | External nameserver responses are untrusted network data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | S | cmd/dns-discovery/main.go | mitigate | Validate and normalize domain input before DNS queries; reject invalid labels early |
| T-01-02 | T | internal/discovery/dns.go | mitigate | Parse DNS answers by expected RR type and ignore malformed/unsupported payloads |
| T-01-03 | D | internal/discovery/dns.go | mitigate | Enforce resolver timeout and continue on non-fatal record lookup failures |
| T-01-04 | R | cmd/dns-discovery/main.go | accept | CLI is local tooling with no multi-user auth context; no repudiation control needed in Phase 1 |
| T-01-05 | I | internal/discovery/providers.go | mitigate | Return only derived provider metadata and raw NS hostnames already public in DNS |
| T-01-06 | E | internal/discovery/providers.go | accept | No privilege model in local CLI; risk not applicable to current architecture |
</threat_model>

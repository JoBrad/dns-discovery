<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| discovery result -> renderer | Untrusted/variable scan data is transformed into output artifacts |
| renderer output -> filesystem | Generated content is written to user-controlled output locations |
| domain-level errors -> stderr/log sink | Failure information crosses user-visible and persistent log channels |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | T | internal/report/output.go | mitigate | Route through explicit output enum switch with no implicit fallback to unknown flavors |
| T-04-05 | R | internal/app/run.go | mitigate | Emit deterministic per-domain success/failure records and preserve input order for auditability |
| T-04-06 | I | stderr/log error surfacing | mitigate | Include domain and operation context in surfaced errors while avoiding raw stack dumps |
</threat_model>
